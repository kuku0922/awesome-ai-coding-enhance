# Vue 3 Project Standards and Directory Layout

This document defines the standard project structure, configuration files, and development standards for Vue 3 applications generated by this skill.

## Official Vue 3 Project Standards

### Based on `create-vue` Official Standards

The project structure follows the official [Vue 3 project scaffolding tool](https://github.com/vuejs/create-vue) standards with additional enhancements for production readiness.

## Directory Structure

### Standard Layout

```
vue3-project/
├── public/                     # Static assets (not processed by Vite)
│   ├── favicon.ico            # Site favicon
│   └── logo192.png            # PWA icon
├── src/                       # Application source code
│   ├── assets/               # Static assets processed by Vite
│   │   └── logo.png
│   ├── components/           # Vue components
│   │   ├── TheWelcome.vue    # Example component
│   │   ├── icons/            # Icon components
│   │   │   └── IconCommunity.vue
│   │   └── __tests__/        # Component tests
│   │       └── TheWelcome.spec.ts
│   ├── router/               # Vue Router configuration
│   │   └── index.ts
│   ├── stores/               # Pinia state management
│   │   └── counter.ts        # Example store
│   ├── views/                # Page components
│   │   ├── AboutView.vue
│   │   ├── HomeView.vue
│   │   └── __tests__/        # View tests
│   ├── App.vue               # Root component
│   ├── main.ts               # Application entry point
│   └── style.css             # Global styles
├── tests/                    # End-to-end tests
│   └── e2e/
│       └── example.spec.ts
├── .env                      # Environment variables
├── .env.development          # Development environment variables
├── .env.production           # Production environment variables
├── .eslintrc.cjs             # ESLint configuration
├── .gitignore                # Git ignore rules
├── .prettierrc               # Prettier configuration
├── .vscode/                  # VS Code configuration
│   ├── extensions.json       # Recommended extensions
│   └── launch.json           # Debug configuration
├── index.html                # HTML template
├── package.json              # Project metadata and dependencies
├── pnpm-lock.yaml            # pnpm lock file (if using pnpm)
├── README.md                 # Project documentation
├── tsconfig.app.json         # TypeScript app configuration
├── tsconfig.json             # TypeScript base configuration
├── tsconfig.node.json        # TypeScript Node.js configuration
└── vite.config.ts            # Vite build configuration
```

### Vite Configuration Standards

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  server: {
    port: 5173,
    host: true
  },
  preview: {
    port: 4173,
    host: true
  },
  build: {
    sourcemap: true,
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor': ['vue'],
          'router': ['vue-router'],
          'store': ['pinia']
        }
      }
    }
  }
})
```

## Code Standards

### Vue Component Standards

1. **Composition API**: Use `<script setup>` syntax with TypeScript
2. **Single File Components**: Follow official Vue SFC structure
3. **Component Naming**: Use PascalCase for component names
4. **Props Definition**: Use TypeScript interfaces for props
5. **Event Definition**: Use TypeScript interfaces for emits

### TypeScript Standards

1. **Strict Mode**: Enable all strict TypeScript checks
2. **Type Definitions**: Create comprehensive type definitions
3. **Import/Export**: Use ES module syntax consistently
4. **Naming Conventions**: Follow TypeScript naming conventions

### CSS Standards

1. **Scoped Styles**: Use scoped styles in components
2. **CSS Variables**: Use CSS custom properties for theming
3. **Responsive Design**: Use mobile-first approach
4. **Accessibility**: Include proper ARIA support

## Testing Standards

### Unit Testing

- **Framework**: Vitest with Vue Test Utils
- **Coverage**: Minimum 80% code coverage
- **Test Files**: Co-located with source files in `__tests__` directories
- **Naming**: Use `.spec.ts` suffix for test files

### Integration Testing

- **Framework**: Vitest for component integration tests
- **Test Scenarios**: User workflows and component interactions
- **Mock Data**: Use consistent mock data across tests

### End-to-End Testing

- **Framework**: Playwright or Cypress
- **Test Scenarios**: Critical user journeys
- **CI/CD**: Automated testing in deployment pipeline

## Performance Standards

### Bundle Size

- **Initial Load**: < 500KB gzipped
- **Code Splitting**: Automatic route-based splitting
- **Tree Shaking**: Eliminate unused code
- **Image Optimization**: WebP format with fallbacks

### Runtime Performance

- **First Contentful Paint**: < 1.5 seconds
- **Largest Contentful Paint**: < 2.5 seconds
- **Cumulative Layout Shift**: < 0.1
- **First Input Delay**: < 100ms

### Development Experience

- **Hot Module Replacement**: Instant development updates
- **Type Checking**: Real-time TypeScript validation
- **Linting**: Automatic code formatting and error detection
- **Build Time**: < 30 seconds for production build

## Security Standards

### Content Security Policy

- **Default Source**: Restrict to trusted sources
- **Inline Scripts**: Avoid inline JavaScript
- **External Resources**: Validate all external resources

### Input Validation

- **Form Validation**: Client and server-side validation
- **XSS Prevention**: Sanitize user input
- **CSRF Protection**: Implement CSRF tokens

### Authentication and Authorization

- **JWT Tokens**: Secure token handling
- **Role-Based Access**: Implement proper authorization
- **Session Management**: Secure session handling

## Deployment Standards

### Build Process

- **Environment Configuration**: Separate dev/staging/production configs
- **Asset Optimization**: Optimize images, fonts, and code
- **Source Maps**: Generate for production debugging
- **Error Tracking**: Integrate error monitoring

### CI/CD Pipeline

- **Automated Testing**: Run all test suites
- **Code Quality**: Enforce linting and formatting
- **Security Scanning**: Check for vulnerabilities
- **Deployment**: Automated deployment to staging/production

### Infrastructure

- **CDN**: Use CDN for static assets
- **HTTPS**: Enforce HTTPS for all requests
- **Browser Caching**: Implement proper caching strategies
- **Monitoring**: Application performance monitoring

This comprehensive set of standards ensures that Vue 3 applications generated by this skill follow industry best practices and are production-ready from day one.
